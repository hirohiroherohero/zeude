## 1. Skills (슬래시 커맨드)

대시보드에서 스킬을 만들면 팀원들의 Claude Code에 `/스킬이름` 커맨드로 자동 배포된다.

### 생성 방법
1. 대시보드 → **Skills** → 스킬 추가
2. 입력:
   - **Name**: 표시 이름 (예: `Code Review`)
   - **Slug**: 커맨드 이름, kebab-case (예: `code-review` → `/code-review`)
   - **Description**: 설명
   - **Content (SKILL.md)**: 스킬 프롬프트 내용
   - **Files**: 멀티파일 스킬의 경우 추가 파일 업로드 가능
3. **Scope**: Global(전체) 또는 특정 팀
4. Save

### 예시

```markdown
# SKILL.md - Code Review 스킬
Review the code in the current file for:
- Bugs and logic errors
- Security vulnerabilities
- Performance issues
- Code style improvements

Give specific line-by-line feedback.
```

### 유저 경험
- 팀원이 Claude Code에서 `/code-review` 입력
- 해당 프롬프트가 자동으로 Claude에 전달됨
- 동기화: `~/.claude/commands/{slug}.md` 또는 `~/.claude/skills/{slug}/`에 저장

## 2. Hooks (이벤트 트리거 스크립트)

Claude Code의 특정 이벤트에 자동으로 스크립트를 실행한다.

### 지원 이벤트

| 이벤트 | 설명 | 활용 예 |
|--------|------|--------|
| `UserPromptSubmit` | 유저가 프롬프트 입력 시 | 프롬프트 로깅, 스킬 추천 |
| `Stop` | 실행 멈출 때 | 세션 종료 로깅 |
| `PreToolUse` | 도구 호출 전 | 위험한 도구 차단 |
| `PostToolUse` | 도구 호출 후 | 결과 감사 로깅 |
| `Notification` | 알림 발생 시 | 외부 알림 전송 |
| `SubagentStop` | 서브에이전트 종료 시 | 서브에이전트 모니터링 |

### 생성 방법
1. 대시보드 → **Hooks** → 추가
2. 입력:
   - **Name**: 훅 이름
   - **Event**: 이벤트 종류 선택
   - **Script Type**: bash / python / node
   - **Script**: 실행할 스크립트
   - **Env**: 환경변수 (선택)
3. Scope 설정 → Save

### 예시 (프롬프트 로거)

```bash
#!/bin/bash
echo "[$(date)] Prompt submitted by $ZEUDE_USER_EMAIL" >> ~/.zeude/prompt.log
```

### 유저 경험
- 훅은 백그라운드에서 자동 실행 (유저에게 보이지 않음)
- 동기화: `~/.claude/hooks/{event}/{hook-name}.sh` 에 저장
- `~/.claude/settings.json`에 자동 등록

### 자동 주입 환경변수

훅 실행 시 다음 환경변수가 자동으로 주입된다:
- `ZEUDE_AGENT_KEY` — 에이전트 키
- `ZEUDE_USER_EMAIL` — 유저 이메일
- `ZEUDE_TEAM` — 소속 팀
- `ZEUDE_API_URL` — 대시보드 API URL

## 3. MCP Servers (외부 도구 연결)

Claude Code에 외부 도구/API를 연결한다.

### 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| **Command** | 로컬 프로세스 실행 | `npx @modelcontextprotocol/server-filesystem` |
| **HTTP** | 원격 MCP 서버 연결 | `https://mcp-fetch.vercel.app/sse` |

### 생성 방법
1. 대시보드 → **MCP Servers** → 추가
2. Command 타입:
   - **Name**: 서버 이름
   - **Command**: 실행 명령어
   - **Args**: 인자 (JSON 배열)
   - **Env**: 환경변수
3. HTTP 타입:
   - **Name**: 서버 이름
   - **URL**: MCP 서버 URL
4. Scope 설정 → Save

### 유저 경험
- `~/.claude.json`의 `mcpServers`에 자동 등록
- Claude Code 시작 시 자동 연결
- 유저가 별도 설정 없이 바로 사용 가능

## 4. Agents (AI 역할 프로필)

Claude의 성격/역할을 커스텀 프로필로 정의한다.

### 생성 방법
1. 대시보드 → **Agents** → 추가
2. 입력:
   - **Name**: 에이전트 이름, kebab-case (예: `security-auditor`)
   - **Files**: 에이전트 프로필 파일 (Markdown)
3. Scope 설정 → Save

### 예시 (보안 감사자)

```markdown
# security-auditor.md
You are a security-focused code auditor.
Always check for: SQL injection, XSS, CSRF,
authentication bypasses, and sensitive data exposure.
Flag issues by severity (Critical/High/Medium/Low).
```

### 유저 경험
- `~/.claude/agents/{name}.md`에 저장
- Claude Code에서 에이전트 전환 가능
- 선택한 에이전트의 프롬프트가 세션에 적용됨

## 5. Team (팀 관리)

### 멤버 초대
1. 대시보드 → **Team** → Invite 버튼
2. 팀 이름, 역할(admin/member) 선택
3. 초대 링크 생성 → 복사하여 팀원에게 전달
4. 팀원이 링크 열면 가입 완료 → agent_key 발급

### 역할

| 역할 | 권한 |
|------|------|
| **admin** | 모든 기능 + Team/Analytics 메뉴 접근 |
| **member** | 대시보드 조회 + 배포된 스킬/훅/MCP 사용 |

## 6. 동기화 메커니즘

### 동작 방식

```
Admin이 대시보드에서 생성/수정
        ↓
  Supabase DB에 저장
        ↓
  팀원이 claude 실행 시 zeude shim이 자동 sync
        ↓
  ~/.claude/commands/   (스킬)
  ~/.claude/hooks/      (훅)
  ~/.claude.json        (MCP)
  ~/.claude/agents/     (에이전트)
```

### 특징
- **빠른 시작**: 캐시된 설정으로 즉시 실행, 백그라운드에서 동기화
- **효율적**: 해시 비교(ETag)로 변경 없으면 전송 스킵
- **오프라인 지원**: 네트워크 끊기면 캐시된 마지막 설정 사용
- **깔끔한 정리**: 삭제된 리소스는 자동으로 로컬에서도 제거
