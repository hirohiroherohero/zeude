# Zeude 데이터베이스 스펙

---

## Supabase (PostgreSQL) — 설정/상태 저장

### 테이블 목록

| 테이블 | 역할 | 주요 컬럼 |
|--------|------|----------|
| `zeude_users` | 유저 계정 | id, email, name, agent_key, team, role, status |
| `zeude_sessions` | 로그인 세션 | token, user_id, expires_at |
| `zeude_one_time_tokens` | OTT 인증 토큰 | token, user_id, expires_at |
| `zeude_invites` | 팀 초대 링크 | token, team, role, created_by, expires_at |
| `zeude_skills` | 스킬 정의 | name, slug, content, files(JSONB), teams, is_global |
| `zeude_hooks` | 훅 스크립트 | name, event, script_content, script_type, env(JSONB) |
| `zeude_mcp_servers` | MCP 서버 설정 | name, command, args(JSONB), env(JSONB), url |
| `zeude_agents` | 에이전트 프로필 | name, description, files(JSONB), max 256KB |
| `zeude_cohort_members` | 코호트 그룹 | user grouping |
| `zeude_skill_rules` | 스킬 추천 규칙 | skill invocation rules |
| `zeude_skill_keywords_tier` | 스킬 키워드 | keyword-based categorization |
| `zeude_hook_install_status` | 훅 설치 추적 | hook_id, agent_id, status |
| `zeude_mcp_install_status` | MCP 설치 추적 | mcp_server_id, agent_id, status |
| `zeude_user_disabled_skills` | 스킬 비활성화 | soft delete tracking |

## ClickHouse — 텔레메트리/분석 저장

### 테이블 목록

| 테이블/뷰 | 역할 | TTL |
|-----------|------|-----|
| `claude_code_logs` | 원본 텔레메트리 (토큰, 비용, 소요시간) | 90일 |
| `ai_prompts` | 프롬프트 이력 (텍스트, 타입, 세션) | 180일 |
| `pricing_model` | 모델별 과금 단가 (Claude, GPT) | - |
| `token_usage_hourly` | 시간별 토큰 사용량 집계 (Materialized View) | 90일 |
| `efficiency_metrics_daily` | 일별 효율 지표 (캐시 히트율, 평균 토큰) | - |
| `retry_analysis` | 재시도 패턴 분석 | - |
| `context_growth_analysis` | 세션 내 입력 토큰 증가 추적 | - |
| `frustration_analysis` | 사용자 좌절 신호 감지 | - |

### claude_code_logs 주요 속성

```
Timestamp, TraceId, SpanId, ServiceName, Body
LogAttributes:
  - user.id, user.email, session.id
  - model, duration_ms
  - input_tokens, output_tokens
  - cache_read_tokens, cache_creation_tokens
  - cost_usd
ResourceAttributes:
  - zeude.user.id, zeude.user.email
```
