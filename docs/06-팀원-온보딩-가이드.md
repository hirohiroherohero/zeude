# Zeude 팀원 온보딩 가이드

## 사전 준비

시작하기 전에 아래 2가지가 설치되어 있어야 합니다.

| 항목 | 확인 방법 | 설치 방법 |
|------|----------|----------|
| **Claude Code** | `claude --version` | `npm install -g @anthropic-ai/claude-code` |
| **jq** | `jq --version` | `brew install jq` (Mac) |

---

## Step 1. 초대 링크 받기

Admin에게 초대 링크를 받습니다. 형태는 아래와 같습니다:

```
https://zeude-vwuo-ehfh2idms-hirohiroheroheros-projects.vercel.app/invite/토큰값
```

---

## Step 2. Agent Key 발급

1. 초대 링크를 브라우저에서 엽니다
2. **이름**과 **회사 이메일**을 입력합니다
3. **"Get Agent Key"** 버튼을 클릭합니다
4. `zd_` 로 시작하는 Agent Key가 화면에 표시됩니다

> **이 키는 한 번만 표시됩니다. 반드시 복사해두세요!**

---

## Step 3. Zeude CLI 설치

아래 명령어를 **한 번에 전체 선택**해서 터미널에 붙여넣으세요:

```bash
export ZEUDE_DOWNLOAD_BASE=https://github.com/hirohiroherohero/zeude/releases/download/v0.1.0
export ZEUDE_DASHBOARD_URL=https://zeude-vwuo-ehfh2idms-hirohiroheroheros-projects.vercel.app
export ZEUDE_ENDPOINT=https://zeude-vwuo-ehfh2idms-hirohiroheroheros-projects.vercel.app
curl -fsSL https://raw.githubusercontent.com/hirohiroherohero/zeude/main/zeude/scripts/install.sh | bash
```

설치 중 `Enter your agent key (zd_xxx):` 라고 물어보면 **Step 2에서 복사한 키를 붙여넣기**하면 됩니다.

### 설치 스크립트가 자동으로 하는 것

- OS/아키텍처 감지 (Mac ARM, Mac Intel, Linux)
- 원본 claude 바이너리 위치 탐색 및 저장
- Zeude shim 바이너리 다운로드 (`~/.zeude/bin/claude`, `~/.zeude/bin/zeude`)
- 설정 파일 생성 (`~/.zeude/config`)
- `ZEUDE_DASHBOARD_URL` 환경변수 등록 (`.zshrc`에 자동 추가)
- Agent Key 저장 (`~/.zeude/credentials`)
- PATH 설정 (zeude shim이 원본 claude보다 먼저 실행되도록)
- `/zeude` 스킬 설치 (대시보드 바로가기)

---

## Step 4. 셸 재시작 및 확인

```bash
# 셸 설정 적용
source ~/.zshrc

# shim이 제대로 설정되었는지 확인
which claude
# 기대 출력: /Users/유저명/.zeude/bin/claude

# 실행 테스트
claude
```

정상이면 아래와 같이 표시됩니다:

```
[zeude] cached, 2 hooks, 3 skills
[zeude] Ready! Hi 유저이름
```

---

## Step 5. 대시보드 접속 (선택)

Claude Code 안에서 `/zeude`를 입력하면 대시보드가 브라우저에서 자동으로 열립니다.

또는 직접 접속:
1. https://zeude-vwuo-ehfh2idms-hirohiroheroheros-projects.vercel.app 접속
2. Agent Key를 이용한 자동 로그인

---

## Zeude 삭제 (Uninstall)

아래 명령어를 터미널에 붙여넣으세요:

```bash
curl -fsSL https://raw.githubusercontent.com/hirohiroherohero/zeude/main/zeude/scripts/uninstall.sh | bash
```

삭제되는 항목:
- `~/.zeude/` 디렉토리 (shim, 설정, credentials)
- Zeude가 설치한 hooks (`~/.claude/hooks/`)
- Zeude가 등록한 MCP 서버 (`~/.claude.json`)
- `/zeude` 스킬 (`~/.claude/commands/zeude.md`)
- 셸 rc 파일의 PATH 설정

삭제 후 셸을 재시작하면 원본 `claude` 명령어가 복원됩니다:

```bash
source ~/.zshrc
which claude  # 원본 경로로 복원 확인
```

## 문제 해결

### `sync failed` 가 뜨는 경우

정상 동작에 영향 없음 (fail-open 설계). 원인은 보통:
- 네트워크 일시 오류 → 다음 실행 시 자동 복구
- 대시보드 서버 일시 다운 → 캐시된 설정으로 동작

### `no agent key` 가 뜨는 경우

Agent Key가 설정되지 않은 상태. 아래 명령어로 수동 설정:

```bash
echo 'agent_key=zd_여기에키붙여넣기' > ~/.zeude/credentials
```

### `claude not found` 에러

Claude Code가 먼저 설치되어 있어야 합니다:

```bash
npm install -g @anthropic-ai/claude-code
```

설치 후 다시 Step 3를 실행하세요.

### 진단 도구

```bash
zeude doctor
```

shim 설치 상태, 원본 claude 경로, 네트워크 연결 등을 자동 점검합니다.

## 설치 후 구조

```
~/.zeude/
├── bin/
│   ├── claude          ← Zeude shim (이것이 먼저 실행됨)
│   └── zeude           ← 진단/관리 도구
├── config              ← 대시보드 URL, endpoint 설정
├── credentials         ← Agent Key 저장 (chmod 600)
└── real_binary_path    ← 원본 claude 경로

실행 흐름:
claude 입력 → ~/.zeude/bin/claude (shim) → 설정 동기화 → 원본 claude 실행
```
