# Zeude 설치 과정 및 시행착오

## 설치 순서 (README 기준)

```
1. Clone repository
2. Install dependencies (pnpm install)
3. Configure environment (.env)
4. Run database migrations (Supabase)
5. Install CLI on client machines
```

---

## 겪었던 시행착오

### 1. .env 파일 위치 문제

- `.env.example`이 프로젝트 루트에 있어 루트에 `.env`를 생성했으나, Next.js 대시보드는 `zeude/dashboard/` 폴더에서 실행되므로 해당 위치에도 `.env`가 필요
- **증상**: 대시보드 API 호출 시 `Internal server error` 발생
- **해결**: `zeude/dashboard/.env`에 복사

### 2. 인증 흐름이 문서에 명확하지 않음

- 대시보드 접속 시 "Authentication Failed" 화면만 표시
- 최초 유저 생성 방법이 README에 안내되어 있지 않음
- **해결**: Supabase SQL Editor에서 직접 INSERT:
```sql
INSERT INTO zeude_users (email, name, agent_key)
VALUES ('email@example.com', 'Name', 'zd_' || encode(gen_random_bytes(32), 'hex'))
RETURNING agent_key;
```

### 3. Admin 권한 설정 누락

- 유저를 생성해도 기본 role이 admin이 아니어서 Team 메뉴가 보이지 않음
- **해결**: SQL로 수동 업데이트:
```sql
UPDATE zeude_users SET role = 'admin' WHERE agent_key = 'zd_xxx...';
```

### 4. CLI 설치 스크립트 로컬 실행 불가

- `curl -fsSL http://localhost:3000/releases/install.sh | bash` → 404 에러
- `/releases/install.sh` 경로는 Docker 배포 시에만 존재 (Dockerfile에서 복사)
- **해결 (Admin)**: Go 바이너리를 직접 빌드 후 수동 복사:
```bash
brew install go
cd zeude && bash scripts/build-release.sh
cp releases/claude-darwin-arm64 ~/.zeude/bin/claude
cp releases/zeude-darwin-arm64 ~/.zeude/bin/zeude
chmod +x ~/.zeude/bin/*
```
- **해결 (팀원 배포용)**: 빌드된 바이너리를 GitHub Releases에 업로드하여 install.sh가 다운로드 가능하도록 함:
```bash
# Admin이 한 번 실행
gh release create v0.1.0 releases/* --repo hirohiroherohero/zeude --title "v0.1.0"
```
- 팀원은 `ZEUDE_DOWNLOAD_BASE`를 GitHub Releases URL로 지정하여 install.sh 실행
- 상세 가이드: `docs/06-팀원-온보딩-가이드.md`

### 7. dashboard_url 미설정으로 sync failed

- CLI 실행 시 `[zeude] sync failed` 표시
- `~/.zeude/config`의 `dashboard_url`이 `http://localhost:3000` (기본값)으로 되어있어 로컬 대시보드가 없으면 동기화 실패
- **해결**: Vercel 배포 URL로 변경:
```
dashboard_url=https://zeude-vwuo.vercel.app
```
- fail-open 설계로 sync 실패해도 claude 사용은 가능하지만, 스킬/훅 동기화가 안 됨

### 5. Vercel 배포 시 환경변수 누락

- `CLICKHOUSE_PASSWORD`가 프로덕션에서 필수로 요구됨
- ClickHouse 없이 PoC할 때도 더미 값 필요
- **해결**: Vercel Environment Variables에 더미 값 추가

### 6. Vercel Deployment Protection

- 기본적으로 Vercel Authentication이 켜져 있어 API 호출이 차단됨
- **해결**: Settings → Deployment Protection → Vercel Authentication Off

## 개선 제안 (오픈소스 기여 가능 포인트)

1. 최초 admin 유저 생성을 위한 seed 스크립트 또는 CLI 명령어 제공
2. 로컬 개발 환경에서의 설치 가이드 보강
3. `.env` 파일 위치에 대한 명확한 안내
4. Go 빌드 환경 prerequisites 명시
5. ClickHouse 없이도 빌드 가능하도록 환경변수 분리
